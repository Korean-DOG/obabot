# Стратегия распространения obabot

## Каналы продвижения (помимо Хабра)

### 1. Telegram-каналы и чаты разработчиков

Это твоя прямая целевая аудитория — люди, которые уже пишут ботов на aiogram.

**Конкретные площадки:**

| Канал/чат | Что делать | Приоритет |
|-----------|-----------|-----------|
| **aiogram chat** (@aiogram_ru, ~15k+ участников) | Написать пост "сделал адаптер, который позволяет запускать ваших aiogram-ботов в Max". Не спамить — один раз, с примером кода | Высокий |
| **Python чаты** (@ru_python, @python_flood) | Короткий пост с wow-эффектом (до/после код) | Средний |
| **Боты и разработка** (профильные чаты про ботов) | Отвечать на вопросы "как запустить бота в Max?" со ссылкой на obabot | Средний |
| **Max-чаты для разработчиков** (если есть) | "Вот как можно быстро портировать Telegram-бота в Max" | Высокий |

**Формат:** Не рекламный пост, а "ребята, я тут сделал штуку, может кому пригодится". С кодом. Без маркетинговых клише.

### 2. GitHub: SEO и видимость

**Что сделать прямо сейчас:**

- **Topics на репозитории:** `telegram-bot`, `max-bot`, `aiogram`, `umaxbot`, `cross-platform`, `bot-framework`, `python`, `async`
- **README на английском** (основной) + русский вариант отдельным файлом. GitHub-аудитория международная, но конкретно по Max — русскоязычная. Держи оба.
- **GitHub Discussions** — включить, чтобы люди могли задавать вопросы не только через issues
- **Awesome-списки:** Есть `awesome-aiogram`, `awesome-telegram-bots`, `awesome-python` — попробовать добавить через PR. Даже если не примут сразу, это видимость.

### 3. Dev.to / Medium (английская аудитория)

Адаптировать хабр-статью на английский. Dev.to бесплатный, статьи индексируются хорошо. Заголовок вроде: "Your aiogram bot now works on Max messenger. Here's how."

Даже если Max пока в основном русскоязычная история — статья на английском даёт SEO и авторитет проекту.

### 4. Reddit

- **r/Python** — "Show /r/Python" пост. Короткий, с примером кода.
- **r/Telegram** — "Built a library to run Telegram bots on multiple platforms"
- **r/coding**, **r/programming** — если пост зайдёт на r/Python

Формат: "I built X because Y. Here's how it works: [код]. GitHub: [ссылка]". Реддит любит самоделки и не любит маркетинг.

### 5. YouTube / видео

Короткое демо-видео (3-5 минут):
1. Открываешь терминал
2. `pip install obabot`
3. Показываешь код бота на aiogram
4. Меняешь 3 строки
5. Запускаешь — показываешь, как бот отвечает в Telegram и Max одновременно

Это самый мощный формат для wow-эффекта. Можно залить на:
- YouTube
- Rutube (для русской аудитории)
- Shorts / Reels (если уложишься в 60 секунд — будет огонь)

### 6. Конференции и митапы

- **PyCon Russia** (или аналог) — lightning talk "Кросс-платформенные боты за 5 минут"
- **Moscow Python** — митап, формат доклада 15-20 минут
- **Онлайн-митапы** — многие сообщества проводят стримы, можно предложить выступление

Даже если не возьмут — попытка стоит. Это авторитет и видимость.

### 7. Документация как маркетинг

Хорошая документация — это канал привлечения сам по себе. Люди гуглят "как запустить бота в Max", находят твою доку, пробуют.

**Что стоит сделать:**
- Отдельный сайт с документацией (MkDocs / Sphinx / Docusaurus) + GitHub Pages
- Гайды: "Миграция с aiogram", "Бот для Max за 5 минут", "FSM на двух платформах"
- SEO: ключевые слова "бот для Max", "aiogram Max", "кросс-платформенный бот Python"

## Стоит ли связываться с платформой Max?

### Короткий ответ: Да, однозначно стоит.

### Длинный ответ:

**Почему это выгодно обеим сторонам:**

Для **Max** (VK Team):
- Они заинтересованы в расширении экосистемы разработчиков. Чем больше ботов — тем живее платформа.
- obabot снижает барьер входа для огромного пула aiogram-разработчиков. Это тысячи потенциальных новых ботов в Max.
- Они получают бесплатный инструмент привлечения разработчиков, который не нужно разрабатывать и поддерживать.

Для **тебя**:
- Официальное упоминание / рекомендация от платформы — это авторитет и трафик.
- Доступ к ресурсам: возможно, ранний доступ к API, поддержка, документация.
- Потенциал: партнёрская программа, упоминание в официальных гайдах, место в каталоге инструментов.

**Как подойти к этому:**

1. **Не приходи с пустыми руками.** Сначала:
   - Опубликуй на PyPI
   - Напиши статью на Хабре
   - Собери хотя бы 20-30 звёзд на GitHub
   - Убедись, что есть реальные пользователи (хотя бы пара)

2. **Найди правильного человека.** Не пиши в общий support. Ищи:
   - DevRel / Developer Advocate команды VK / Max
   - Технического менеджера бот-платформы Max
   - Через LinkedIn, через знакомых, через Telegram-чаты разработчиков Max

3. **Питч должен быть про них, не про тебя:**
   > "Привет! Я написал open-source библиотеку, которая позволяет разработчикам aiogram-ботов запускать своих ботов в Max за 5 минут, меняя 3 строки кода. Это может привлечь в Max тысячи существующих aiogram-ботов. Вот демо, вот GitHub, вот статья на Хабре. Было бы здорово, если вы упомянете obabot в документации для разработчиков или в блоге."

4. **Что просить:**
   - Упоминание в официальной документации Max для разработчиков ботов
   - Пост в официальном блоге / канале для разработчиков
   - Совместный вебинар / митап
   - Фидбек по API — что ещё нужно поддержать

5. **Чего НЕ просить (пока):**
   - Денег
   - Эксклюзивности
   - Найма

**Тайминг:** Лучше после публикации на Хабре и PyPI. Приходить с готовым продуктом и социальным доказательством (звёзды, статья, пользователи) — это совсем другой разговор, чем приходить с идеей.
